### 1. Executive Summary

### 2. Problem Statement & Opportunity

*   **[Brief/PRD]:** Clearly define the problem(s) this project aims to solve.
    *   *Example:* "Users struggle to find relevant information quickly on our existing platform, leading to high bounce rates and customer support queries."
*   **[Brief/PRD]:** What is the underlying pain point for users or the business?
*   **[Brief/PRD]:** What is the market opportunity or strategic advantage this project unlocks?
    *   *Example:* "Addressing this problem will improve user retention, reduce operational costs, and increase conversion rates by X%."

### 3. Vision & Goals

*   **[Brief/PRD] Vision:** What is the long-term desired state or ultimate impact of this project? Where are we trying to go?
    *   *Example:* "To create the most intuitive and personalized experience for our users, making information discovery effortless."
*   **[Brief/PRD] Goals (SMART Objectives):** What specific, measurable, achievable, relevant, and time-bound goals will define success for *this particular project*?
    *   *Example:*
        *   Increase user engagement (time on site/app) by 15% within 3 months post-launch.
        *   Reduce customer support tickets related to information discovery by 20% within 6 months.
        *   Improve conversion rate for Feature X by 5% in Q4.

### 4. Target Audience / Users

*   **[Brief/PRD]:** Who are we building this for?
    *   *Primary Users:* [e.g., "Existing Customers (SMB owners) aged 30-55," "New User Onboarding," "Internal Sales Team"]

*   **[Brief/PRD]:** Briefly describe their key needs, pain points, or motivations relevant to this project.
*   **[PRD]:** Link to or include user personas if available.

### 5. Solution Overview & Key Features (MVP)

*   **[Brief/PRD] Solution Overview:** Describe the proposed solution at a high level. What are we building?
    *   *Example:* "A new intelligent search functionality combined with a personalized content recommendation engine."
*   **[Brief/PRD] Minimum Viable Product (MVP):** What is the absolute minimum set of features required to launch and achieve the core goals?
    *   *List core features/components.*
*   **[PRD] Detailed Features / User Stories:**
    *   Break down the solution into specific features.
    *   For each feature, list user stories (as a [user role], I want to [action], so that [benefit]).
    *   *Example:*
        *   **Feature: Intelligent Search**
            *   *User Story 1:* As a business owner, I want to search for invoices by date range, so that I can quickly reconcile my accounts.
            *   *User Story 2:* As an admin, I want to see trending search terms, so that I can identify common user needs.
        *   **Feature: Personalized Recommendations**
            *   *User Story 1:* As a new user, I want to see recommended articles based on my onboarding preferences, so that I can learn about relevant features faster.
*   **[PRD] User Flows/Journey:** Describe the key user flows. (Can be text description or link to diagrams/wireframes).
